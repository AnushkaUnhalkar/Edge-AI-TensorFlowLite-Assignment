Q1. Define Edge AI. Explain how it is different from Cloud AI. (5 Marks)
-Edge AI
Edge AI refers to artificial intelligence processing that happens locally on a device (edge device) such as smartphones, IoT devices, cameras, or Raspberry Pi instead of sending data to a cloud server
The AI model runs directly on the device where data is generated.
-Difference Between Edge AI and Cloud AI:
| Feature             | Edge AI                 | Cloud AI                     |
| ------------------- | ----------------------- | ---------------------------- |
| Processing Location | On local device         | Remote cloud server          |
| Latency             | Very low                | Higher (depends on internet) |
| Internet Required   | Not required            | Required                     |
| Privacy             | High (data stays local) | Lower (data sent to server)  |
| Computing Power     | Limited                 | Very high                    |


Q2. What is TensorFlow Lite? Why is it used in Edge devices? (5 Marks)
-TensorFlow Lite (TFLite)
TensorFlow Lite is a lightweight version of TensorFlow designed to run machine learning models on mobile and embedded devices.
-Why it is used in Edge Devices:
Small model size
Low memory usage
Fast inference
Optimized for ARM processors
Supports hardware acceleration (GPU, Edge TPU)
-It allows AI models to run efficiently on devices like smartphones and Raspberry Pi.


Q3. Explain Model Compression. What are: (5 Marks)
-Model Compression
Model compression reduces the size and complexity of AI models so they can run efficiently on edge devices.

1.Quantization
Quantization reduces model precision.
-Example:
Convert 32-bit floating-point numbers (float32)
To 8-bit integers (int8)
-Benefits:
Smaller model size
Faster inference
Lower power consumption

2.Pruning
Pruning removes unnecessary weights (connections) from the neural network.
-Benefits:
Reduces model size
Improves speed
Keeps important parameters only
